C#中的表达式树

Expression Trees
CSharp

在学EFCore的又看到如下语句

IQueryable<T> Query<T>(Expression<Func<T, bool>> funcWhere) where T : class;

先来看两个简单的例子

使用场景：EF/AutoMapper/ORM框架

表达式树以树形数据结构（二叉树）表示代码，其中每一个节点都是一种表达式，比如方法调用和 x < y 这样的二元运算等。

它将我们原来可以直接由代码编写的逻辑以表达式的方式存储在树状的结构里，从而可以在运行时去解析这个树，然后执行，实现动态的编辑和执行代码。在.Net 里面的Linq to SQL就是对表达式树的解析。

表达式的创建

根据 Lambda 表达式创建表达式树

Func<int, bool> func = num => num < 5;
Console.WriteLine(func(4));

Expression<Func<int, bool>> func_exp = num => num < 5;
//这种方法创建出的表达式根节点类型为ExpressionType.Lambda，Type类型为返回值类型typeof(bool)
Console.WriteLine(func_exp(4));	//这段代码将报错

C# 编译器只能从表达式 Lambda（或单行 Lambda）生成表达式树。 它无法解析语句 lambda （或多行 lambda）。

我们直接输出Console.WriteLine(func_exp);

返回的结果是num => (num < 5)

如果我们想要编译表达式树并运行结果代码

我们就需要运行Console.WriteLine(func_exp.Compile()(4));

也可以写成Console.WriteLine(func_exp.Compile().Invoke(4));

又或者var func_cp= func_exp.Compile(); Console.WriteLine(func_cp(4));

就会返回结果True

LambdaExpression类和Expression<TDelegate>类都有一个Compile的方法，学名是Lambda表达式的委托，其实就是Lambda表达式编译函数的委托，所以我们只需要调用他，得到的结果就是一个函数方法。

通过 API 创建表达式树

ParameterExpression numParam = Expression.Parameter(typeof(int), "num");
ConstantExpression five = Expression.Constant(5, typeof(int));
BinaryExpression numLessThanFive = Expression.LessThan(numParam, five);
Expression<Func<int, bool>> lambda1 =
    Expression.Lambda<Func<int, bool>>(
        numLessThanFive,
        new ParameterExpression[] { numParam });

我们创建了一个参数表达式num，类型为int

一个常量表达式five，类型为int

一个具有二进制运算符的表达式，将num和five组装在一起

最终的表达式为num < 5

我们直接输出Console.WriteLine(lambda1);

返回的结果是num => (num < 5)

表达式树的构造

Expression<TDelegate>的功能是将强类型Lambda表达式表示为表达式树形式的数据结构，他的父类是LambdaExpression，比较他们代码可知，Lambda表达式的主体，名称和参数全部保存在LambdaExpression里面。

以下两张图是旧的，但是大致差不多

LambdaExpression里面的Body就是我们的表达式。

C#表达式给我们提供了丰富的表达式类，进入到LambdaExpression类里面

方法返回类型以“Expression”结尾的，基本上都是一个表达式类。

常用表达式

ConstantExpression：常量表达式

ParameterExpression：参数表达式

UnaryExpression：一元运算符表达式

BinaryExpression：二元运算符表达式

TypeBinaryExpression：is运算符表达式

ConditionalExpression：条件表达式

MemberExpression：访问字段或属性表达式

MethodCallExpression：调用成员函数表达式

Expression<TDelegate>：委托表达式

解析表达式树

Expression<Func<int, bool>> func_exp = num => num < 5;

ParameterExpression param = (ParameterExpression)func_exp.Parameters[0];
BinaryExpression operation = (BinaryExpression)func_exp.Body;
ParameterExpression left = (ParameterExpression)operation.Left;
ConstantExpression right = (ConstantExpression)operation.Right;

Console.WriteLine("Decomposed expression: {0} => {1} {2} {3}",

手动拼接表达式树

手动拼接表达式可以写出非常复杂的表达式。

（）=》 5

现在有一个非常简单的表达式()=>5，我们如何进行手动拼接呢？

Expression<Func<int>> funcExp = () => 5;

首先我们可以去LINQPad中查看表达式树，如下

  • 第一步根据Body最内层创建表达式

我们知道表达式类型为ConstantExpression，常量表达式

节点类型为Constant，需要用到Expression.Constant

Value为5

ConstantExpression constExp=Expression.Constant(5);

  • 第二部根据外层创建表达式

返回类型为Expression<Func<int>>

节点类型为Lambda，需要用到Expression.Lambda

需要一个Body（表达式）还有参数，在这里Body为ConstantExpression的表达式，然后没有参数，可以填null

Expression<Func<int>> funcExp = Expression.Lambda<Func<int>>(constExp,null);

综合起来就是：

ConstantExpression constExp=Expression.Constant(5);
Expression<Func<int>> funcExp = Expression.Lambda<Func<int>>(constExp,null);
var result = funcExp.Compile(); //进行编译并保存到result
Console.WriteLine(result());

y => (5 + y)

现在有一个非常简单的表达式y => (5 + y)，我们如何进行手动拼接呢？

Expression<Func<int, int>> funcExp = (y) => 5 + y;

也是一样去LINQPad中查看表达式树，如下

  • 从内向外写，首先是Left

我们知道表达式类型为ConstantExpression，常量表达式

节点类型为Constant，需要用到Expression.Constant

Value为5，类型为Int

ConstantExpression constantExpression = Expression.Constant(5,typeof(int));

  • 然后是Right

我们知道表达式类型为ParameterExpression，参数表达式

节点类型为Parameter，需要用到Expression.Parameter

参数名称为y，类型为int

ParameterExpression parameterExpression = Expression.Parameter(typeof(int),"y");

  • 之后我们再回到Body

我们知道表达式类型为BinaryExpression，二元操作符表达式或者叫二元运算符表达式，也就是有两个操作数

节点类型为Add，需要用到Expression.Add

操作数为Left和Right中的常量和参数

BinaryExpression binaryExpression = Expression.Add(constantExpression, parameterExpression);

  • 最后是最外层表达式

返回类型为Expression<Func<int,int>>

节点类型为Lambda，需要用到Expression.Lambda

需要一个Body（表达式）还有参数，在这里Body为BinaryExpression的表达式，然后有一个参数，就是parameterExpression

Expression<Func<int, int>> funcExp = Expression.Lambda<Func<int, int>>(binaryExpression,parameterExpression);

ConstantExpression constantExpression = Expression.Constant(5,typeof(int));
ParameterExpression parameterExpression = Expression.Parameter(typeof(int),"y");
BinaryExpression binaryExpression = Expression.Add(constantExpression, parameterExpression);
Expression<Func<int, int>> funcExp = Expression.Lambda<Func<int, int>>(binaryExpression,parameterExpression);
var result = funcExp.Compile();
Console.WriteLine(result(5));

Expression<Func<int, int>> funcExp = Expression.Lambda<Func<int, int>>(binaryExpression,parameterExpression);也可以写成

Expression<Func<int, int>> funcExp = Expression.Lambda<Func<int, int>>(binaryExpression,new ParameterExpression[] { parameterExpression});

(y, x) => ((5 + (y * y)) - x)

接下来来一个复杂一点的

Expression<Func<int,int,int>> funcExp = (y,x) => 5+y*y-x;

也是一样去LINQPad中查看表达式树，如下

通过上述图示也是可以很简单的写出表达式树，具体过程就不在描述了

ConstantExpression five = Expression.Constant(5,typeof(int));
ParameterExpression y = Expression.Parameter(typeof(int), "y");
BinaryExpression yMuly = Expression.Multiply(y, y);
BinaryExpression FiveAddyMuly= Expression.Add(five, yMuly);
ParameterExpression x = Expression.Parameter(typeof(int), "x");
BinaryExpression FiveAddyMulySubx=Expression.Subtract(FiveAddyMuly, x);
Expression<Func<int, int, int>> funcExp = Expression.Lambda<Func<int, int, int>>(FiveAddyMulySubx,new ParameterExpression[] { y, x});
var result = funcExp.Compile();
Console.WriteLine(result(2,3)); //结果为6

(x) => x.Id;

有如下代码，将(x) => x.Id手动拼接表达式树

Expression<Func<Student, object>> funcExp = (x) => x.Id;
funcExp.Dump();

class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
}

也是一样去LINQPad中查看表达式树，如下

写出来也比较简单

ParameterExpression x =Expression.Parameter(typeof(Student), "x");
MemberExpression xid = Expression.Property(x, "Id");    //x.id
UnaryExpression unaryExpression = Expression.Convert(xid, typeof(object));
Expression<Func<Student, object>> funcExp = Expression.Lambda<Func<Student, object>>(unaryExpression, x);
var result = funcExp.Compile();
Console.WriteLine(result(new Student { Id=5,Name="8yyy"}));

需要注意的是：

MemberExpression属于访问字段或属性表达式

  • 如果你要访问属性就使用Expression.Property
  • 如果你要访问字段就使用Expression.Field

UnaryExpression 属于一元操作符表达式或者一元运算符表达式

上述表达式树中，只需要将MemberExpression xid = Expression.Property(x, "Name"); //x.Name

就能输出Student类的另一个属性Name，结果为8yyy

表达式树创建自动对象转换器

例如有两个类，他们的成员完全一致，但是不是同一类型

class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
}

class NewStudent
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
}

按照正常思路，手动转换

static void Main(string[] args)
{
    Student student= new Student() { 
        Id= 8,
        Name="8yyy",
        Age= 8,
    };
    NewStudent newStudent = new NewStudent()
    {
        Id = student.Id,
        Name = student.Name,
        Age = student.Age,
    };
}

但是一旦成员过多，这样写的效率就很低，在现实开发中用的比较多的是AutoMapper自动映射框架

同时这种转换也可以用序列化反序列化和反射来做，但是效率不如表达式树

首先创建一个ExpressionTreesExample类，代码如下

using System;
using System.Collections.Generic;
using System.Linq;
using System.Linq.Expressions;
using System.Text;
using System.Threading.Tasks;

namespace ExpressionTreesExample
{
    public class ExpressionMapper<Tin, Tout>
    {
        public static Func<Tin, Tout> func;
        static ExpressionMapper()
        {
            
        }
        private static Tout Mapper(Tin t)
        {
            ParameterExpression parameterExp = Expression.Parameter(typeof(Tin), "p"); //Tin类型的参数p 
            List<MemberBinding> memberBindings = new List<MemberBinding>();
            //绑定属性
            foreach (var item in typeof(Tout).GetProperties())
            {
                MemberExpression member = Expression.Property(parameterExp, typeof(Tin).GetProperty(item.Name)); //p.Id,p.Name,p.Age
                MemberBinding memberBinding = Expression.Bind(item, member);
                //将Tout的成员与p.Id,p.Name,p.Age分别绑定
                memberBindings.Add(memberBinding);
            }
            //绑定字段
            foreach (var item in typeof(Tout).GetFields())
            {
                MemberExpression member = Expression.Field(parameterExp, typeof(Tin).GetField(item.Name)); //p.(item.Name)
                MemberBinding memberBinding = Expression.Bind(item, member);
                memberBindings.Add(memberBinding);
            }
            //创建对象并初始化
            MemberInitExpression memberInitExpression = Expression.MemberInit(Expression.New(typeof(Tout)), memberBindings.ToArray());

            Expression<Func<Tin, Tout>> funcExp = Expression.Lambda<Func<Tin, Tout>>(memberInitExpression, new ParameterExpression[] { parameterExp });
            func = funcExp.Compile();
            return func(t);
        }
    }
}

之后回到Program.cs

static void Main(string[] args)
{
    Student student = new Student() {
        Id = 8,
        Name = "8yyy",
        Age = 8,
    };
    NewStudent newStudent = ExpressionMapper<Student, NewStudent>.Mapper(student);
    Console.WriteLine($"{newStudent.Id},{newStudent.Name},{newStudent.Age}");
}

注释：

MemberBinding 类提供表示绑定的类派生自的基类，这些绑定用于对新创建对象的成员进行初始化。

Expression.Bind 方法 创建一个表示成员初始化的 MemberAssignment。

MemberInitExpression 类 表示调用构造函数并初始化新对象的一个或多个成员。

类似于([Extent1].[Id] > 2) AND ([Extent1].[Address] LIKE '%杭州%')

为此我们需要了解ExpressionVisitor 类

ExpressionVisitor 类表示表达式树的访问者或重写者。

其中Visit是一个主管，你给它一个表达式，它会进行分配，比如说二元表达式会分配给VisitBinary

常量表达式会分配给VisitConstant，成员表达式会分配给VisitMember

Ctrl + M + O: 折叠所有方法

Ctrl + M + M: 折叠或者展开当前方法

Ctrl + M + L: 展开所有方法

具体代码如下：

using System;
using System.Collections.Generic;
using System.ComponentModel.Design;
using System.Linq;
using System.Linq.Expressions;
using System.Text;
using System.Threading.Tasks;

namespace EFExample.LinqToSql
{
    public class ConditionVisitor:ExpressionVisitor
    {
        private Stack<string> stringStack= new Stack<string>();
        
        //返回栈里的条件字符串
        public string Condition()
        {
            string condition = string.Concat(stringStack.ToArray());
            stringStack.Clear();
            return condition;
        }
        //([Extent1].[Id] > 2) AND ([Extent1].[Address] LIKE '%杭州%')
        //二元表达式
        protected override Expression VisitBinary(BinaryExpression node)
        {
            stringStack.Push(")");
            base.Visit(node.Right); //解析右边
            stringStack.Push(" "+node.NodeType.ToSQLOperator()+" ");
            base.Visit(node.Left); //解析左边
            stringStack.Push("(");
            return node;
        }
        //([Extent1].[Id] > 2) AND ([Extent1].[Address] LIKE '%杭州%')
        //成员表达式
        protected override Expression VisitMember(MemberExpression node)
        {
            stringStack.Push($" [{node.Member.Name}] ");
            return node;
        }
        //([Extent1].[Id] > 2) AND ([Extent1].[Address] LIKE '%杭州%')
        //常量表达式
        protected override Expression VisitConstant(ConstantExpression node)
        {
            Type type = node.Value.GetType();
            if(type.Equals(typeof(string))|| type.Equals(typeof(DateTime)))//如果是字符或者日期，需要加单引号
            {
                stringStack.Push($"{node.Value}");//成员入栈不要有空格
            }
            else
            {
                stringStack.Push(node.Value.ToString());
            }
            return node;
        }
        //([Extent1].[Id] > 2) AND ([Extent1].[Address] LIKE '%杭州%')
        //方法表达式
        protected override Expression VisitMethodCall(MethodCallExpression node)
        {
            Visit(node.Object);//左边：方法所在的对象，返回是成员表达式--》跳转到VisitMember
            Visit(node.Arguments[0]);//右边：方法参数，返回常量表达式--》VisitConstant

            //获取入栈数据，为方法解析做参数（参数和字段名）
            string right=stringStack.Pop(); //'杭州'
            string left = stringStack.Pop();    //Adress

            switch (node.Method.Name)
            {
                case "Contains":
                    stringStack.Push($"{left} LIKE '%{right.Replace(",","")}%'");
                    break;
                case "Equals":
                    stringStack.Push($"{left}={right}");
                    break;
                default:
                    throw new NotSupportedException(node.NodeType+"不支持！");
            }
            return node;
        }
    }
}

SQLOperator类用于将表达式树节点的节点类型转换成SQL运算符

using System;
using System.Collections.Generic;
using System.Linq;
using System.Linq.Expressions;
using System.Text;
using System.Threading.Tasks;

namespace EFExample.LinqToSql
{
    internal static class SQLOperator
    {
        public static string ToSQLOperator(this ExpressionType type)//扩展方法
        {
            switch (type)
            {
                case ExpressionType.Add:
                case ExpressionType.AndAlso:
                    return "AND";
                case ExpressionType.OrElse:
                case ExpressionType.Or:
                    return "OR";
                case ExpressionType.Not:
                    return "NOT";
                case ExpressionType.NotEqual:
                    return "<>";
                case ExpressionType.GreaterThan:
                    return ">";
                case ExpressionType.GreaterThanOrEqual:
                    return ">=";
                case ExpressionType.LessThan:
                    return "<";
                case ExpressionType.LessThanOrEqual:
                         return "<=";
                case ExpressionType.Equal:
                    return "=";
                default:
                    throw new Exception("不支持该方法");
            }
        }
    }
}

表达式树的永久性

表达式树应具有永久性。 这意味着如果你想修改某个表达式树，则必须复制该表达式树然后替换其中的节点来创建一个新的表达式树。 你可以使用表达式树访问者遍历现有表达式树。
