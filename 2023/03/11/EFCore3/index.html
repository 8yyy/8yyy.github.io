在EF Core中混用Database First和Code First

EF Core DB First Code First
.NET
发布日期:   2023-03-11
文章字数:   1.2k
阅读时长:   4 分

前言

在EF Core中有两种主要的设计工作流(Design WorkFlows)：

  • Code First：创建类（POCO 实体）并根据它们生成新的数据库。
  • Database First：使用现有的数据库并根据数据库模式生成类。

其中 Database First 从现有的数据库**逆向工程(Reverse Engineering)**生成模型；

Code First 添加新的类或修改现有的类，并使用**迁移(Migrations)**来更新数据库。

无论是逆向工程还是迁移都是由Microsoft.EntityFrameworkCore.Tools提供的。

Add-Migration 开启迁移 (一般用法：Add-Migration 迁移名称)

Bundle-Migration

Drop-Database

Get-DbContext

Get-Migration

Optimize-DbContext

Remove-Migration

Scaffold-DbContext 上下文脚手架

Script-Migration

Update-Database

其中逆向工程需要用到Scaffold-DbContext命令;

迁移需要用到Add-Migration以及Update-Database 等命令。

在实际项目开发过程中，一般我们都推荐使用Code First模式，即代码先行的原则。

如果你已经创建好了数据库也可以使用Database First模式。但是随着项目的推进，我们仍然可能需要修改实体关系，或者新增字段，或者删除某个字段，亦或者增加新的表等。这时我们如果继续采取逆向工程，通过Scaffold-DbContext命令这种方式，虽然很简单，但可能会覆盖之前对模型的更改，引起丢失之前设置的数据注解或者Fluent API配置等等其他的问题。（如果是非常简单的问题，例如添加字段，我们也可以同时更改实体/上下文与数据库保持一致即可）

这时候我们就可以选择使用Code First中的迁移。

打开程序包管理器控制台，使用Scaffold-DbContext命令

Scaffold-DbContext "Data Source=.\sqlexpress;Initial Catalog=test2;Integrated Security=True" Microsoft.EntityFrameworkCore.SqlServer -OutputDir Entity -ContextDir Context -Context testDbContext -Force

其中"Data Source=.\sqlexpress;Initial Catalog=test2;Integrated Security=True"是数据库连接字符串，Microsoft.EntityFrameworkCore.SqlServer为数据库提供程序，-OutputDir指定用于输出类的目录（即实体的目录），-ContextDir用于指定上下文类的目录，-Context用于指定上下文名称，-Force强制覆盖现有文件。

使用迁移

打开程序包管理器控制台，使用Add-Migration命令

Add-Migration AddTableHello

其中AddTableHello为迁移的名称，因为我们是要新建一张名为Hello的表，迁移名称的用途与版本控制系统中的提交消息类似。

在项目文件下生成了一个Migrations文件夹

其中20230311054553_AddTableHello.cs 主迁移文件(The main migrations file)，包含应用迁移所需的操作（在 Up 中）和还原迁移所需的操作（在 Down 中）。

20230311054553_AddTableHello.Designer.cs迁移元数据文件(The migrations metadata file)，包含 EF 所用的信息。

testDbContextModelSnapshot.cs当前模型的快照(A snapshot of your current model)， 用于确定添加下一迁移时的更改内容。

可以看到迁移已经添加成功，注意这时候我们不能直接使用Update-Database 命令直接将修改应用到数据库。原因是使用先使用Database First时，数据库中不存在__MigrationHistory表，系统认为需要重新创建所有表，实际上people表已经存在于数据库里了。
